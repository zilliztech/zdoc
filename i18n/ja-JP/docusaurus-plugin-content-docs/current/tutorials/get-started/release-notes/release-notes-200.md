# リリースノート (2023年6月11日)

Zilliz Cloudのリリースは、ベクトルデータベース管理に新たな標準を設定します。初心者のユーザーエクスペリエンスを大幅に向上させ、より手頃で柔軟な価格オプションを提供し、シームレスなチームコラボレーションを可能にし、柔軟なスキーマ管理を提供します。このアップデートの主な機能には、サーバーレスクラスター、多様なティアプラン、組織およびコラボレーションサポート、RBACサポート、パーティションキー、動的スキーマ、およびJSON型サポートが含まれます。今日からこの画期的なアップデートをお試しください！

## Milvus互換性\{#milvus-compatibility}

このリリースは**Milvus 2.1.x**と互換性があります。

## Zilliz Cloudサーバーレスベクトルデータベースサービスのローンチ\{#zilliz-cloud-serverless-vector-database-service-launch}

Zilliz Cloudのサーバーレスベクトルデータベースサービスのローンチを発表できることを嬉しく思います。この新オファリングは、専用クラスターソリューションと比較して、さらに手軽なベクトルデータ検索エクスペリエンスを提供します。

初期段階のベンチャーを支援するために、Serverless Serviceの一部として2つの無料コレクションを提供しています。各コレクションは、768次元規模の50万ベクトルを処理する容量を備えており、広範なインフラを必要とせずに実質的なデータ処理能力を提供します。私たちの革新的な新機能をお試しになり、今日からデータ処理能力を向上させてください。[無料トライアル](./free-trials)。

## Zilliz Cloudのプランティア：Starter、Standard、Enterprise、およびSelf-hosted\{#zilliz-clouds-plan-tiers-starter-standard-enterprise-and-self-hosted}

Starter、Standard、Enterprise、およびSelf-hostedの4つのプランティアをご提供できることを嬉しく思います。各ティアは、コスト、サービス、およびセキュリティの考慮事項のバランスを取るように設計されており、すべてのユーザーにニーズに合ったプランがあります。Zilliz Cloudを使用することで、ビジネスの成長に合わせてベクトルデータベース管理を柔軟にスケールおよび進化させる柔軟性があります。

- Starterプラン：Serverlessインスタンスで完全に管理されたベクトルデータベースを体験するための最も簡単な方法です。最小限の設定が必要で、GCPで利用可能です。

- Standardプラン：5人未満のエンジニアおよび複雑なワークロードを持つチーム向けに設計されています。高度なデータベース機能および構成を持つ専用クラスターを提供し、優れたコスト効率を実現します。AWSおよびGCPの両方で利用可能です。

- Enterpriseプラン：大規模な組織で高度なセキュリティおよびサポートが必要な場合に設計されています。すべての機能、高可用性、高度な構成を備えた専用クラスターを提供し、AWSおよびGCPの両方で堅牢なデータベースソリューションを提供します。

- Self-hostedプラン：プライバシーと規制遵守を優先するビジネスに最適です。仮想プライベートクラウド（VPC）内で自己管理型ベクトルデータベースサービスを提供し、完全な制御を可能にします。プライバシーと規制遵守を優先する環境に適しています。

[価格ページで詳細を確認してください。](https://zilliz.com/pricing)

## 組織、コラボレーション、およびRBAC\{#organization-collaboration-and-rbac}

Zilliz Cloudが高度な組織およびメンバー管理機能を提供することを発表できることを嬉しく思います。これにより、複数のユーザーがクラスター、プロジェクト、および組織のさまざまなレベルでシームレスにコラボレーションできます。

このメジャーアップデートは、ロールベースアクセス制御（RBAC）の統合によってサポートされており、特定のリソースに誰がアクセスできるかを細かく制御できます。これにより、セキュリティと柔軟性の両方が確保され、チームは高いデータ保護を維持しながらより効率的に作業できます。

この機能強化は、プロジェクトアクセスに対するより大きな制御を提供するだけでなく、コラボレーションプロセスも合理化し、チームがより安全で整理された環境でベクトルデータベースを管理しやすくします。今日からこの機能を試し、Zilliz Cloudでの協調効率の新しいレベルを体験してください。詳細については、[アクセス制御](./access-control)を参照してください。

## パーティションキー\{#partition-key}

Zilliz Cloud 2.0.0は、パーティションキーフィーチャーを導入します。この機能により、コレクションを作成する際に特定のフィールドをパーティションキーとして指定できます。エンティティは、そのパーティションキーの値に基づいてパーティションに格納されます。

この機能はクエリフィルタリング時特に有用であり、パーティションキーフィールドの条件は従来のスキャンベースのフィルタリング方法よりも大幅に高速に実行できます。実質的に、同じパーティションキーを持つエンティティは物理的にグループ化されているため、不要なスキャンを回避できます。

## 動的スキーマ\{#dynamic-schema}

Zilliz Cloudがバージョン2.0.0から動的スキーマをサポートすることを発表できることを嬉しく思います。このメジャーエンハンスメントは、多様なビジネス要件に対応する柔軟性を大幅に向上させます。ユーザーは、事前に定義された静的スキーマに限定されることなく、動的に変化するフィールドを持つエンティティをコレクションに挿入できるようになりました。

上級ユーザー向けに、動的および静的スキーマを融合したコレクション作成のハイブリッドアプローチを組み込みました。これにより、ユーザーはスキーマ設計で「必須」フィールドを指定し、高度なインデックス最適化を有効にすることができます。一方で、「オプション」フィールドは動的スキーマメカニズムでサポートされます。この革新的なアプローチは、クエリ性能を向上させながらスキーマの柔軟性を維持します。

Zilliz Cloud 2.0.0でこの機能を探索し、データベーススキーマの適応性に大幅な向上を体験してください。[この機能について詳細を確認してください](./enable-dynamic-field)。

## JSON型サポート\{#json-type-support}

最新アップデートでのZilliz Cloudのメジャーなエンハンスメント、JSONデータ管理および近似最近傍（ANN）検索機能の統合を発表できることを嬉しく思います。

JSON（JavaScript Object Notation）は、今日のデータベース管理の世界で不可欠になった重要なデータ交換形式です。Zilliz CloudへのJSONサポートの導入により、JSONデータを簡単に保存および管理できるようになり、データ操作およびクエリの可能性が広がります。

真の力は、既存のANN検索機能との革新的な統合にあります。この組み合わせにより、JSONデータ構造の柔軟性とANN検索の精度を組み合わせた複雑なクエリを実行できるようになり、データベースクエリ機能において大きな飛躍を遂げます。

これらの2つの強力な技術の相乗効果を1か所で体験し、データ管理およびクエリタスクでの効率と精度の新たなレベルを解放してください。今日からZilliz Cloudでデータベーステクノロジーの未来に進みましょう。[この機能について詳細を確認してください](./use-json-fields)。
