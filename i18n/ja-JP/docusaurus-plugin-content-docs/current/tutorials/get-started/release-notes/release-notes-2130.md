# リリースノート（2025年1月27日）

Zilliz Cloudは現在、**Public Preview**でMilvus 2.5をサポートし、セマンティック検索を補完するフルテキスト検索を導入しています。BM 25メトリックとスパースベクトルを使用して効率的なストレージと取得を行うことで、この機能により、ユーザーは変換なしでテキストを直接クエリできます。このアップデートにより、BYOCデプロイメントがPrivate Linkで強化され、セキュリティが向上し、コンプライアンスが容易になり、ネットワーク構成が簡素化されました。さらに、AWS CloudFormationを介した自動デプロイメントがサポートされ、大規模な小さなファイルのインポートの処理が最適化され、より高速なデータ取り込みが可能になりました。

## Milvusの互換性{#milvus}

このリリースは**Milvus v 2.4. x**と互換性があります。

クラスターを**パブリックプレビュー**にアップグレードしたい場合は、アップグレード後に**Milvus 2.5. x**の機能を利用できます。

## Zilliz Cloudは、Milvus 2.5のパブリックプレビューをサポートし、全文検索を導入しました{#zilliz-cloudmilvus-25}

Zilliz Cloudは現在、**パブリックプレビュー**として利用可能なMilvus 2.5と互換性があります。このアップデートでは、主要な新機能である**全文検索**(レキシカルまたはキーワード検索とも呼ばれます)が導入されました。検索が初めての場合、全文検索により、Googleで検索するのと同様に、特定の単語やフレーズを検索してドキュメントを見つけることができます。これは、単に正確な単語を一致させるのではなく、意味を理解することに焦点を当てた既存の**意味検索**機能を補完します。

私たちの実装は、文書の類似性に対して**業界標準BM 25**メトリックを使用し、**疎ベクトル**に基づいており、効率的なストレージと検索を可能にしています。簡単に言えば、疎ベクトルは、ほとんどの値がゼロである方法でテキストを表現します。数個のセルに数字が含まれ、残りは空である大規模なスプレッドシートを想像してください。この効率性は、ベクトルが検索の基盤として機能するMilvusのコア哲学に合致しています。

Zillizのアプローチの主な利点は、事前に手動でスパースベクトルに変換することな**く、テキストを直接挿入およびクエリできることです。**これにより、Zilliz Cloudは非構造化データ処理を完全にサポートするためにさらに一歩近づきました。

詳細については、[フルテキスト検索](./full-text-search)を参照してください。

## BYOC-強化されたセキュリティと簡単な展開{#byoc}

このリリースにより、**Zilliz CloudはPrivate Linkを介したデータプレーン-コントロールプレーン通信をサポート**し、BYOCデプロイメントの**より強力なセキュリティとコンプライアンス**を提供します。

- **強力なセキュリティ**:プライベートリンクにより、コントロールプレーン（Zilliz Cloudにホストされている）とデータプレーン（VPCにデプロイされている）間のすべての通信が**プライベートネットワーク内**にとどまり、パブリックインターネットを完全にバイパスします。これ**により、サイバー脅威への曝露を減ら**し**、データ傍受のリスクを排除します**。

- **容易なコンプライアンス**:多くの企業は、規制基準を満たすために**厳格なデータ居住地とネットワーク分離**を必要としています。プライベートリンクを使用すると、**機密データがプライベート環境を離れな**いようにすることができ、**GDPR、HIPAA、およびその他のセキュリティフレームワーク**に準拠しやすくなります。

- **簡素化されたネットワーク構成**: Private Linkにより、**複雑なファイアウォールルール、VPN、またはパブリックエンドポイント**が不要になり、運用オーバーヘッドが削減され、BYOCの展開が**より簡単かつ管理しやすく**なります。

さらに、**Zilliz Cloud BYOCはAWS CloudFormationを介した自動デプロイをサポート**し、データプレーンの設定と管理がさらに簡単になりました。

詳細については、[BYOCをAWSにデプロイ](./deploy-byoc-aws)を参照してください。
