# リリースノート（2025年1月27日）

Zilliz Cloudは現在、**Public Preview**でMilvus 2.5をサポートしており、セマンティック検索を補完するフルテキスト検索を導入しています。BM 25メトリックとスパースベクトルを使用して効率的なストレージと取得を行うことで、この機能により、ユーザーは変換なしでテキストを直接クエリできます。このアップデートにより、BYOCデプロイメントがPrivate Linkで強化され、セキュリティが向上し、コンプライアンスが容易になり、ネットワーク構成が簡素化されました。さらに、AWS CloudFormationを介した自動デプロイメントがサポートされ、大規模な小さなファイルのインポートの処理が最適化され、より高速なデータ取り込みが可能になりました。

## Milvusの互換性{#milvus-compatibility}

このリリースは**Milvus v 2.4. x**と互換性があります。

クラスターを**パブリックプレビュー**にアップグレードしたい場合は、アップグレード後に**Milvus 2.5. x**の機能が利用可能です。**パブリックプレビュー**の機能について詳しく知るには、Zilliz Cloudコンソールの**クラスターの詳細**ページで**プレビュー機能を試す**をクリックしてください。

## Zilliz Cloudは、Milvus 2.5のパブリックプレビューをサポートし、全文検索を導入しました{#zilliz-cloud-now-supports-milvus-25-in-public-preview-introducing-full-text-search}

Zilliz Cloudは現在、**パブリックプレビュー**として利用可能なMilvus 2.5と互換性があります。このアップデートでは、主要な新機能である**全文検索**、またはレキシカル検索またはキーワード検索としても知られています。検索が初めての場合、全文検索により、Googleで検索するのと同様に、特定の単語やフレーズを検索してドキュメントを見つけることができます。これは、単に正確な単語を一致させるのではなく、意味を理解することに焦点を当てた既存の**意味検索**機能を補完します。

私たちの実装は、文書の類似性に対して業界標準のBM 25メトリックを使用し、疎ベクトルに基づいているため、効率的なストレージと検索が可能です。簡単に言えば、疎ベクトルは、ほとんどの値がゼロである方法でテキストを表現します。数個のセルに数字が含まれ、残りは空である大規模なスプレッドシートを想像してください。この効率性は、ベクトルが検索の基盤として機能するMilvusのコア哲学に合致しています。

Zillizのアプローチの重要な利点は、事前に手動でスパースベクトルに変換する必要がなく、テキストを直接挿入してクエリできることです。これにより、Zilliz Cloudは非構造化データ処理を完全にサポートするためにさらに一歩近づくことができます。

詳細については、[フルテキスト検索](./full-text-search)を参照してください。

## BYOC-強化されたセキュリティと簡単な展開{#byoc-enhanced-security-and-simplified-deployment}

このリリースにより、Zilliz CloudはPrivate Linkを介したデータプレーン-コントロールプレーン通信をサポートし、BYOCデプロイメントに対してより強力なセキュリティとより簡単なコンプライアンスを提供します。

- 「より強力なセキュリティ」:プライベートリンクにより、コントロールプレーン(Zilliz Cloudにホストされている)とデータプレーン(VPCに展開されている)間のすべての通信が、パブリックインターネットを完全にバイパスして、プライベートネットワーク内にとどまるようになります。これにより、サイバー脅威への露出が減少し、データ傍受のリスクが排除されます。

- **より簡単なコンプライアンス**:多くの企業は、規制基準を満たすために**厳格なデータ居住地とネットワーク分離**を必要としています。Private Linkを使用すると、**機密データがプライベート環境から出ないようにすることができ、**GDPR、HIPAA、およびその他のセキュリティフレームワーク**に準拠することが容易になります。

- 簡素化されたネットワーク構成:プライベートリンクにより、複雑なファイアウォールルール、VPN、またはパブリックエンドポイントが不要になり、運用オーバーヘッドが削減され、BYOCの展開がより簡単かつ管理しやすくなります。

さらに、Zilliz Cloud BYOCは現在、AWS CloudFormationを介した自動展開をサポートしており、データプレーンの設定と管理がさらに簡単になっています。

詳細については、[AWSでBYOCをデプロイする](/docs/byoc/deploy-byoc-aws)を参照してください。

## エンハンスメント{#enhancements}

大規模な小さなファイルのインポートのサポート:多数の小さなファイルを含むインポートの処理が改善され、より速く効率的なデータ取り込みが確保されました。
