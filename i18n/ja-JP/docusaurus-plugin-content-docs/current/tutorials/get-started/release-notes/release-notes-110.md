# リリースノート (2023年4月6日)

コスト推定のための価格計算機、データセキュリティ強化のためのデータバックアップおよびリストアプロセス、グローバルユーザーのためのカスタマイズ可能なタイムゾーン設定、より良い整理のためのコレクション名変更ツール、無制限使用のためのストレージクォータの削除、非アクティブデータベースのクレジット節約改善を特徴とする最新のZilliz Cloudアップデートを発表できることを嬉しく思います。

- 価格計算機

    新機能「価格計算機」のローンチを発表できることを嬉しく思います。この新機能は、ユーザーがコストを効果的に計画および推定するのを支援するように設計されています。価格計算機を使用すると、ユーザーはCUタイプを選択し、エンティティ数とベクトルの次元数を入力して、Zilliz Cloudの推奨CUサイズとコスト推定を簡単に取得できます。[こちらをクリック](https://zilliz.com/pricing#calculator)してお試しください。

    当社の価格計算機は現在ベクトルデータのみをサポートしており、将来はスカラーデータを含むよう機能を拡張し、ユーザーにより正確なコスト推定を提供する予定です。

- GCPのバックアップとリストア

    このリリースにより、当社のプラットフォームはGoogle Cloud Platform (GCP) 上のデータベースバックアップおよびリストアをサポートするようになりました。最新のリリースにより、ユーザーはGCP上でデータベースを簡単にバックアップおよびリストアできるようになり、データ復旧とビジネスの継続性を確保できます。

    詳細については、[バックアップとリストア](./backup-and-restore)を参照してください。

- カスタムタイムゾーン

    タイムゾーン設定のサポートを追加し、ユーザーがタイムゾーン設定をカスタマイズし、すべてのデータのタイムスタンプを正確に保つことを可能にしました。

    詳細については、[タイムゾーン管理](./organization-settings#manage-timezone)を参照してください。

- コレクション名変更

    最新のリリースに、ユーザーがデータベースコレクションの名前を変更できる新機能が含まれていることを嬉しく思います。この機能は、ユーザーがデータベース構造をより適切に管理し、新しいコレクションを作成してデータを移行することなく必要な変更を行うのを支援するように設計されています。

- ストレージクォータの削除

    このリリースにより、当社はプラットフォームからストレージクォータを削除しました。これにより、ユーザーはストレージ制限を心配することなく、空のコレクションにデータをインポートできるようになりました。データストレージには制限がありませんが、各データベースのロード容量はデータベースのサイズに依存します。適切なデータベースサイズを選択するには、価格計算機の使用を強くお勧めします。

    さらに、事前に割り当てられたCUにデータベースを適合させられない場合のユーザー通知も改善しました。ユーザーのデータベースが容量を超えていると通知を受け取ります。これにより、ユーザーはデータのインポートが失敗した理由をより理解し、適切な対応を取ることができます。

- 非アクティブデータベースのクレジット節約の改善

    非アクティブデータベースのプロダクトロジックおよびリマインダ通知を最適化しました。ユーザーのデータベースが14日間連続して非アクティブな場合、無料クレジットを節約するためにユーザーのためにデータベースを自動的に停止します。データセキュリティは常に当社の最優先事項であるため、非アクティブデータベースからデータを削除することはありません。データベースが停止されたユーザーは、いつでも再開できます。また、この最適化を含めるために利用規約も更新しました。

- その他の改善

    - 請求書のUIを改善。
    - CUタイプの命名を更新 - 「High Performance」は「Performance-optimized」に、「Big Data」は「Capacity-optimized」に名称変更。
