# リリースノート (2024年7月23日)

このアップデートでは、Zilliz CloudはMilvusの新しいRESTful API v2をサポートするようになりました。一貫したインターフェースと拡張機能を提供します。新しいドキュメントチャットボットがユーザーサポートを強化します。**ジョブセンター**は、バックアップ、リストア、移行、インポート、およびコレクションのクローンなどのタスクを管理および追跡するための直感的なインターフェースを導入します。**専用クラスター向けの自動スケーリング**はプライベートプレビューで利用可能で、Compute Unit（CU）容量しきい値によってトリガーされ、需要に基づいて容量を動的に調整します。その他の強化には、より多くのクラスターモニタリングメトリクス、刷新されたクラスターマネジメントインターフェース、および改善されたユーザーメールテンプレートが含まれます。

## Milvus互換性\{#milvus-compatibility}

このリリースは**Milvus 2.3.x**と互換性があります。

クラスターをBETAにアップグレードしたい場合は、**Milvus 2.4.x**機能はアップグレード後に利用可能です。

### RESTful API v2\{#restful-api-v2}

最近のMilvus 2.4アップデートで、新しいRESTful API v2がローンチされました。このリリースにより、Zilliz CloudはこれらのAPIを完全にサポートするようになり、一連のコントロールプレーンインターフェースを提供します。新しいv2 APIは、インターフェーススタイルがより一貫しており、v1と比較してより広範な機能をカバーしています。これらの機能には、データプレーンでのベクトル操作、コレクション管理、インデックス管理、パーティション管理、ロールおよびユーザー管理、およびエイリアス操作が含まれます。コントロールプレーンでは、APIはデータインポートおよびクラスターマネジメントをカバーします。詳細については、RESTful v2の[コントロールプレーンAPI](/reference/restful/control-plane-v2)および[データプレーンAPI](/reference/restful/data-plane-v2)を参照してください。

### チャットボット\{#chatbot}

Zilliz Cloudは、従来の検索バーと比較してより柔軟で強力なサポートツールであるドキュメントチャットボットを新たに搭載しました。このチャットボットにより、ユーザーは情報を簡単に検索し、質問に関するサポートを受けることができます。Zilliz Cloudドキュメントページの右下隅にあるアイコンをクリックしてチャットボットにアクセスできます。

### ジョブセンター\{#job-center}

Zilliz Cloudは、単一プロジェクト内のすべての履歴および非同期データタスクを統合する直感的なジョブページを提供するようになりました。この合理化されたインターフェースにより、以下を含むさまざまなタイプのジョブの進行状況を簡単に追跡および管理できます：

- バックアップ

- リストア

- 移行

- インポート

- コレクションのクローン

詳細については、[プロジェクトジョブの管理](./job-center)を参照してください。

### 専用クラスター向けの自動スケーリング [プライベートプレビュー]\{#auto-scaling-for-dedicated-clusters-private-preview}

Zilliz Cloudは、需要に基づいてクラスターの容量を動的に調整する機能である自動スケーリングを導入します。自動スケーリングは主にCU（Compute Unit）容量しきい値によってトリガーされます。Zilliz CloudはクラスターのCU容量を毎分監視し、連続して2分間70%（デフォルトしきい値）を超えると、システムは自動的にスケーリングプロセスを開始します。ユーザーは自動スケーリングの最大CUサイズを設定できますが、現在のところ下方向への自動スケーリングはサポートされていません。

自動スケーリングは現在プライベートプレビューで、専用（エンタープライズ）クラスターでのみ利用可能です。この機能を有効にするには、[お問い合わせください](https://zilliz.com/contact-sales?_gl=1*y9u24o*_ga*NDAwNDA1MDY5LjE3MDkxNTcwNzU.*_ga_KKMVYG8YF2*MTcyMTcwNjA5MC4xMjQuMS4xNzIxNzA5OTk3LjAuMC4w*_ga_HT329313WV*MTcyMTcwNjA5MC4zNS4xLjE3MjE3MDk5OTcuMC4wLjA.*_ga_Q1F8R2NWDP*MTcyMTcwNjA5MC4zMy4xLjE3MjE3MDk5OTcuMC4wLjA.*_gcl_au*ODIwMjEwMjY0LjE3MTcwNjEwOTc.)。使用方法については、[自動スケーリング](./manage-cluster)を参照してください。

### パイプライン\{#pipelines}

- パイプラインは、新しいSEARCH_IMAGE_BY_TEXT機能によりテキストによる画像検索をサポートするようになりました。この機能により、ユーザーはテキストクエリを入力することでデータベースから関連する画像データを検索できます。検索機能は複数言語をサポートし、テキストおよび画像エンコーディングにCLIP vit base patch32マルチモーダルモデルを使用します。

- ユーザーは、RestFul APIおよびUIコンソールの両方を使用して、パイプライン詳細でパイプライン使用情報を取得できるようになりました。この強化により、ユーザーはより包括的なパイプライン使用状況を把握し、より良い監視および分析が可能になります。

- 各プロジェクト内の各タイプのパイプラインの最大数制限が増加しました。従来の10個の制限から、ユーザーは単一プロジェクト内で各タイプのパイプラインを最大100個まで作成できるようになりました。この変更により、プロジェクト内でのパイプライン管理における柔軟性とスケーラビリティが向上します。

### 機能強化\{#enhancements}

このリリースには、一連の機能強化も含まれます：

- クラスター監視のためのより多くの[メトリクス](/docs/metrics-alerts-reference)

- クラスター管理ページのリファクタリング（クラスター変更、移行、バックアップを含む）

- ユーザーメールテンプレートの洗練
