        ## Milvus v2.6.x パブリックプレビュー\{#milvus-v26x-public-preview}

        このリリースにより、**Milvus v2.6.xクラスター**がZilliz Cloudの**パブリックプレビュー**で利用可能になり、安定性、効率、および柔軟性を向上させる複数の強化および最適化が導入されます。

        - **ダウンタイムなしのフィールド追加** — スキーマの回避策なしにコレクションに新しいフィールドをオンザフライで追加できます。詳細については、[既存コレクションへのフィールド追加](./add-fields-to-an-existing-collection)を参照してください。

        - **強化された全文検索** — Elasticsearchよりも**最大4倍高速**で、多言語サポートおよびフレーズ一致機能を備えています。詳細については、[多言語アナライザー](./multi-language-analyzers)、[フレーズ一致](./phrase-match)、および[ユースケースに適したアナライザーの選択](./choose-the-right-analyzer-for-your-use-case)を参照してください。

        - **加速されたJSONフィルタリング** — **JSONインデックス**および**shredding**により、複雑なネストされたメタデータクエリを**最大100倍高速**に実行できます。詳細については、[JSONインデックス作成](./json-indexing)および[JSON Shredding](./json-shredding)を参照してください。

        - **新しい再ランク付け関数** — **ブーストランカー**および**デケイランカー**は、セマンティック類似性と文脈的関連性を組み合わせて検索結果を洗練します。詳細については、[ブーストランカー](./boost-ranker)および[デケイランカー](./decay-ranker)を参照してください。

        - **INT8ベクトルサポート** — 軽量なディープラーニング推論用の量子化ベクトルを保存します。詳細については、[密ベクトル](./use-dense-vector)を参照してください。

        - **MINHASH_LSHインデックス** — MinHashおよびローカリティセンシティブハッシュによって駆動される大規模な重複排除および類似性チェックを効率的に実行します。この機能は**プライベートプレビュー**で利用可能であり、ご興味がある場合は[お問い合わせください](https://support.zilliz.com/hc/en-us)。詳細については、[MINHASH_LSH](./minhash-lsh)を参照してください。

        - **部分アップサート** — 全レコードの書き直し無しに特定のフィールドを更新します。詳細については、[エンティティのアップサート](./upsert-entities#upsert-in-merge-mode)を参照してください。

        **パブリックプレビュー**を有効にするには、Zilliz Cloudコンソールの**クラスタ概要**ページでクラスターをMilvus v2.6.xにアップグレードし、**プレビューフィーチャーを試す**を選択します。アップグレード後も、Milvus v2.5.x機能は引き続き利用可能です。

        ## レイヤードストレージのアップグレード\{#tiered-storage-upgrade}

        Zilliz Cloudのレイヤードストレージは、パフォーマンスとコスト効率を最適化するためにアップグレードされました。すべての拡張容量クラスターは新しいアーキテクチャに移行され、以下の主要改善点が導入されました：

        - **スマートデータ管理**：アクセスパターンに基づいてホット（メモリ）、ウォーム（SSD）、およびコールド（オブジェクトストレージ）レイヤー間でデータを自動的に移動し、パフォーマンスとコスト効率の両方を向上させます。

        - **より高いキャッシュヒット率**：90%以上のキャッシュヒット率で、ほとんどのクエリは高速なレイヤーから提供されます。

        - **コスト削減**：コンピュートコストは25%削減され、ストレージコストはGBあたり月額$0.30から$0.04に87%低下します。10TBのデータセットでは、月額ストレージコストが$3,000から$400に低下し、高パフォーマンスを維持します。

        ## クロスリージョンバックアップ\{#cross-region-backup}

        Zilliz Cloudは、専用クラスター向けにクロスリージョンバックアップをサポートするようになり、災害復旧能力を強化します。この機能は、完全なクラウドリージョン故障に対する耐性を、バックアップを他のリージョンに自動的にレプリケートすることで確保します。

        **主要機能**

        - **自動レプリケーション**：一度バックアップポリシーを構成すれば、Zilliz Cloudが選択した宛先リージョンへの継続的なレプリケーションを自動的に処理します。

        - **地理的冗長性**：オリジナルバックアップから物理的に分離された場所にバックアップコピーを保存して、リージョン故障から保護します。

        - **迅速な復旧**：クロスリージョンバックアップから新しいクラスターにデータをすばやく復元し、ダウンタイムを最小限に抑え、回復時間目標（RTO）を大幅に改善します。

        詳細については、[他のリージョンへのコピー](./backup-to-other-regions)を参照してください。

        ## インデックス構築レベル\{#index-build-level}

        Milvus 2.6.xおよび次世代量子化エンジンにより、検索精度（リコール）とデータ容量とのトレードオフをアプリケーションのニーズに合わせて微調整できます。Zilliz Cloudの新しいインデックス構築レベル機能により、インデックス作成時のベクトル検索パフォーマンスを制御でき、3つのレベルから選択できます：

        - **精度優先**：精度が最優先のミッションクリティカルアプリケーション向けに検索精度を最大化します。

        - **バランス型（デフォルト）**：リコール、パフォーマンス、および容量の理想的なバランスを提供する、ほとんどのユースケースにおすすめの設定。

        - **容量優先**：データ密度を最適化し、クエリリコールを減らしますが、予算内でより多くのベクトルを保存できます。

        詳細については、[インデックス構築レベルの調整](./tune-index-build-level)を参照してください。

        ## 機能強化\{#enhancements}

        - 現在、**アナライザGUI**を使用して、**言語固有のテンプレート**でアナライザをすばやく構成し、結果を**テスト**できます。これにより、ユーザーはアナライザ構成がトークナイズにどのように影響するか（最終的には全文検索結果に影響する）を理解できます。デモンストレーションについては、[アナライザ概要](./analyzer-overview#example-use-on-the-zilliz-cloud-console)を参照してください。

        - より明確なエラーメッセージおよび強化されたエクスペリエンスにより、ユーザーが**接続問題を診断**し、移行用のソースデータベースをより簡単にセットアップできるようになりました。

        - データなしでコレクションをクローンする場合、スキーマを編集し、コレクション設定を変更できるようになりました。

        ## 非推奨通知\{#deprecation-notice}

        - Pipeline機能は非推奨となり、現在はオフラインになっています。
