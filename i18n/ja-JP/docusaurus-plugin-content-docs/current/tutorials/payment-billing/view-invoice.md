# 請求書

Zilliz Cloudは組織レベルで請求します。請求書にアクセスするには、**組織オーナー**または**請求管理者**の権限が必要です。

このガイドでは、請求書の表示、支払い、ダウンロード方法、および請求書詳細の解釈方法について説明します。

<Admonition type="info" icon="📘" title="注意">

<p>マーケットプレイスで登録した場合、Zilliz Cloudの使用量に関する請求書はマーケットプレイス経由で届きます。</p>

</Admonition>

## 請求書の理解\{#understand-your-invoices}

各請求書はいくつかの重要な要素で構成されています。このセクションでは、例として請求書を使用して各要素を理解するお手伝いをします。

![example-invoice](/img/example-invoice.png)

### 請求周期\{#billing-cycle}

請求周期は請求書の上部に表示され、料金が計算された期間と支払い期日を示します。

![Vp6Rwz3Eph1IuXbQgKScVcSEnZg](/img/Vp6Rwz3Eph1IuXbQgKScVcSEnZg.png)

- **請求周期：** 前月の1日00:00:00（UTC）から当月の最終日23:59:59（UTC）までの通常1か月間の期間。たとえば、2024年9月1日にZilliz Cloudは8月の請求書を発行し、請求期間は2024年8月1日00:00:00（UTC）から2024年8月31日23:59:59（UTC）までとなります。この期間中は使用量に応じた料金が蓄積され、請求書のステータスは「**請求前**」のままです。

- **発行日：** 請求書が生成された日付。この日、請求書のステータスが「**未払い**」に変わり、支払いが可能になります。支払い方法（例：クレジットカードまたはマーケットプレイス登録）が追加されている場合、自動的に請求されます。支払いが成功すると、請求書のステータスは「**支払い済み**」に更新されます。支払いが失敗した場合、通知メールが**組織オーナー**および**請求管理者**に送信されます。

- **期日：** 支払いの最終日。この日までに支払いが受け付けられない場合、請求書は**猶予期間**に入ります。

- **猶予期間：** 支払いが可能な14日間の期間。この期間中、毎日メールリマインダーが送信され、支払いが成功するまで請求書のステータスは「**未払い**」のままです。

- **延滞日：** 支払いが未払いの場合、請求書のステータスが「**延滞**」になります。組織が翌日に凍結される可能性があるため、迅速な支払いを推奨します。凍結から1日以内に支払いがない場合、すべてのクラスター（サーバーレスおよび専用）が自動的に[ごみ箱](./use-recycle-bin)に移動され、30日間保持されます。

### 請求書の概要\{#invoice-summary}

概要セクションでは、請求書の料金の高レベルな概要を提供します。

- **使用量金額：** すべての課金対象アイテムの月額合計（CU、ストレージ、バックアップ、パイプライン、および読み書きコストを含む）。

- **クレジット：** 支払いに適用されたクレジット。

- **小計：** 小計 = 使用量金額 - クレジット。

- **税金：** 税金 = 小計 x 税率。税率は請求先住所の国に基づきます。

- **合計金額：** 合計金額 = 小計 + 税金。

- **前払い：** 支払いを相殺するために使用された前払い金額。

- **請求金額/支払金額：** 支払う必要がある最終金額または支払った金額。

### クラスタープラン別概要\{#summary-by-cluster-plan}

Zilliz Cloudは3つのクラスタータイプを提供します：無料、サーバーレス、および専用。料金はサーバーレスおよび専用クラスターのみに適用されます。

- **専用クラスター：** 使用量に基づいて請求されます。料金は `クラスターコスト = クラスタCUサイズ x 実行時間 x 単価` として計算されます。サーバーレスクラスターとは異なり、専用クラスターでは読み書き操作がなくても専用リソース割当のために料金が発生します。

    <Admonition type="info" icon="📘" title="注意">

    <p>専用クラスターコストの場合、実行時間はクラスターの状態が「<strong>実行中</strong>」、「<strong>変更中</strong>」、「<strong>凍結</strong>」などの期間として定義されます。以下の4つの状態では料金は発生しません：「<strong>作成中</strong>」、「<strong>中断中</strong>」、「<strong>再開中</strong>」、または「<strong>中断済み</strong>」。</p>

    </Admonition>

- **サーバーレスクラスター：** 読み書き操作中のvCU消費量に基づく従量課金制で請求されます。料金は `読み書きコスト = vCU使用量 x vCU単価` として計算されます。操作が発生しない場合、ストレージ料金のみが請求されます。

追加料金には以下が含まれます：

- **バックアップコスト：** `バックアップファイルサイズ x バックアップ保持期間` として計算され、「GB-月」で測定されます。これは、1か月間保持された1 GBのバックアップファイルの使用量を指します。**バックアップは、短い保持期間であっても最低1日で請求されます。** つまり、バックアップファイルが作成されても1日未満保持された場合でも、1日の料金で請求されます。

- **ストレージコスト：** `現在のストレージサイズ x クラスターランタイム` として計算され、「GB-時間」で測定されます。これは、1時間保管された1 GBのデータの使用量を指します。**ストレージは、短い保管期間であっても最低1時間で請求されます。**

    <Admonition type="info" icon="📘" title="注意">

    <p>ストレージコストの場合、実行時間はクラスターの状態が「<strong>実行中</strong>」、「<strong>変更中</strong>」、「<strong>凍結</strong>」などの期間として定義されます。以下の状態のクラスターには料金は発生しません：「<strong>作成中</strong>」。</p>

    </Admonition>

### 請求プロファイル\{#billing-profile}

請求プロファイルには、請求書が発行される場所と宛先の詳細が含まれます。Zilliz Cloudでは、関連する請求書メールは組織オーナー、組織請求管理者、および請求プロファイルに追加されたメールアドレスに送信されます。したがって、請求書の受信者を追加するには、請求プロファイルにメールアドレスを追加するか、ユーザーを[招待](./organization-users)して組織に加入させ、組織請求管理者としての役割を与えてください。

請求プロファイルを編集するには、[クレジットカードを追加して登録](./subscribe-by-adding-credit-card#edit-billing-profile)を参照してください。

## トラブルシューティング / よくある質問\{#troubleshooting-faq}

1. **請求書の開始日と終了日はいつですか？**

    **説明：** 請求期間は前月の1日00:00:00（UTC）に開始し、当月の最終日23:59:59（UTC）に終了します。

    **例：** 2024年9月1日にZilliz Cloudは8月の請求書を発行し、請求期間は2024年8月1日00:00:00（UTC）から2024年8月31日23:59:59（UTC）までとなります。

1. **Zilliz Cloudの使用状況詳細に表示される金額はどの程度正確ですか？**

    Zilliz Cloudは**10小数点**まで精度で料金を計算し、すべての請求はこの精度レベルで計算されます。日次料金は最初に10小数点まで計算され、その後請求プロセス中に合計され、10小数点に四捨五入されます。

    - **RESTful API**：すべての数値（例：単価、使用量、使用金額）は常に正確に10小数点で返されます。値が10小数点未満の場合、末尾にゼロが追加されて10桁になります。RESTful APIの使用方法の詳細については、[日次使用状況をクエリ](/reference/restful/query-daily-usage-v2)を参照してください。

    - **Web Console UI**：表示される金額はAPI値と一貫性がありますが、可読性のために末尾のゼロは省略されます。たとえば、`0.1234000000`はUIでは`0.1234`として表示されます。

1. **請求書が届かないのはなぜですか？**

    **考えられる原因：** **組織オーナー**または**請求管理者**のみが請求書にアクセスできます。

    **解決策：** 必要な権限があることを確認してください。請求書にアクセスできない場合は、組織オーナーまたは請求管理者に連絡してください。

1. **支払い方法が失敗した場合どうなりますか？**

    **考えられる原因：** 提供された支払い方法（例：クレジットカード）が期限切れになっているか、資金が不足している可能性があります。

    **解決策：** 支払いが失敗した場合、Zilliz Cloudは**組織オーナー**および**請求管理者**にメールで通知します。組織オーナーおよび請求管理者は、**請求プロファイル**ページで組織の支払い方法を更新し、**14日間の** **猶予期間**内に支払いを再試行できます。

1. **猶予期間とは何ですか？**

    **説明：** **猶予期間**は支払い期日の14日間の期間で、請求書が延滞になる前に支払いができる期間です。

    **ヒント：** この期間中、毎日メールリマインダーが届き、支払いが完了するまで請求書のステータスは未払いのままです。

1. **延滞日以降に支払いをしない場合どうなりますか？**

    **説明：** **猶予期間**内に支払いがされない場合：

    - **延滞日**に、請求書は延滞としてマークされます。

    - **延滞日**の翌日、組織が**凍結**され、Zilliz Cloudサービスへのアクセスが制限されます。

    - 組織が凍結された後1日以内に支払いが行われない場合、すべてのクラスター（サーバーレスおよび専用）が自動的に削除されます。

    **解決策：** サービスの中断とデータ損失を避けるため、**延滞日**までに支払いを解決してください。

1. **サーバーレスクラスターで操作がなくてもなぜ請求されますか？**

    **説明：** サーバーレスクラスターで読み書き操作が発生しなくても、ストレージには請求されます。ストレージコストは、保存されているデータのサイズとZilliz Cloudに保持されている時間を基に計算されます。

    **解決策：** ストレージコストを最小限に抑えるには、未使用のデータを削除することを検討してください。

1. **組織が凍結された旨のメールが届きました。どうすればよいですか？**

    **説明：** 組織が凍結されたことを示すメールを受け取った場合、支払いが延滞しており、Zilliz Cloudサービスへのアクセスが制限されていることを意味します。

    **解決策：**

    組織の凍結を解除するには：

    - クラスターが自動的に削除されないように、凍結後1日以内に必要な支払いを行ってください。

    - 支払いが処理されると、組織の凍結が解除され、クラスターへの完全なアクセスが復元されます。

1. **延滞請求書により自動的に削除されたクラスターをどのようにして復元できますか？**

    **説明：** クラスターが自動的に削除された場合、組織が凍結された後も支払いに失敗したことを意味します。

    **解決策：**

    自動的に削除されたクラスターを復元するには、

    - 最初に支払いをして組織の凍結を解除してください。

    - 支払いが成功すると、ごみ箱に移動した削除されたクラスターを復元できます。

    **ヒント：**

    - 削除されたクラスターはごみ箱に30日間保持されます。クラスターが必要な場合は、クラスター削除後30日以内に延滞した支払いを行ってください。

    - 支払いまたはクラスターの復元中に問題がある場合は、[サポートチケットを作成](http://support.zilliz.com)してください。
