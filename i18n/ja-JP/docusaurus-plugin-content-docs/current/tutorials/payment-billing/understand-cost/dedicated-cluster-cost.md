# 専用クラスターコスト

Zilliz Cloudの専用クラスターは後払いモデルに従い、主にクラスターによって消費されるコンピューティングリソースに対して請求されます。これにより、事前にリソースを過剰にプロビジョニングする必要なく、実際に使用した分だけ料金を支払うことができます。

専用クラスターの総コストは、以下の要素の合計です：

- [ベクターデータベースコスト](./dedicated-cluster-cost#vector-database-cost)

- [ストレージコスト](./dedicated-cluster-cost#storage-cost)

- [その他のコスト](./dedicated-cluster-cost#other-costs)（例：監査ログコストおよびデータ転送料）

## ベクターデータベースコスト\{#vector-database-cost}

### 料金計算\{#cost-calculation}

```plaintext
ベクターデータベースコスト = クエリCU単価 × クエリCUの総数 × クラスターラ行時間
```

- **クエリCU単価**：クラスターリリジョン、タイプ、およびプロジェクトプランによって決定されます。詳細な料金については、[Zilliz Cloud価格](http://zilliz.com/pricing)を参照してください。

- **クエリCUの総数**：レプリカを考慮したクラスター内のクエリCUの総数。

    ```plaintext
    クエリCUの総数 = クエリCU数 × レプリカ数
    ```

    たとえば、2つのクエリCUと2つのレプリカを持つクラスターは、合計4つのCUを持ちます。

- **クラスターラ行時間**：クラスターが課金対象状態になっている総時間（時間単位）：

    - 課金対象状態：実行中、変更中など。

    - 非課金対象状態：作成中、一時停止中、再開中、一時停止中など。非課金対象状態では、CU料金は停止しますが、ストレージ料金は引き続き適用されます。

### 例\{#example}

クラスター構成が以下のようになっていると仮定します：

- **プロジェクトプラン：** エタープライズ

- **クラスターデプロイメントオプション：** 専用

- **クラウドプロバイダーおよびリージョン：** AWS us-east-1（バージニア）

- **クラスタータイプ：** パフォーマンス最適化CU

- **クエリCU数：** 8 CU

- **レプリカ数：** 2

- **クラスターラ行時間：** 720時間（1ヶ月）。

プラン、クラウドプロバイダーおよびリージョン、およびクラスタータイプ情報に基づいて、[価格ページ](https://zilliz.com/pricing)でCU単価が**&#36;0.248/時間**であることがわかります。

![find-cu-unit-price](/img/find-cu-unit-price.png)

クエリCU数およびレプリカ数に応じて、クエリCUの総数は `8 CU x 2レプリカ = 16 CU` です。

例にあげた専用クラスターの合計ベクターデータベースコストは `#0.248 x 16 x 720 = #2856.96` です。

## ストレージコスト\{#storage-cost}

ストレージコストはCUコストとは別に請求され、以下の要因に依存します：

- クラスターコラウドプロバイダーおよびリージョン、タイプ、およびプラン

- ストレージ使用量

詳細については、[ストレージ](./storage-cost)を参照してください。

## その他のコスト\{#other-costs}

アドオン機能は別途請求されます。アドオンコストの発生源は以下の通りです：

- **監査ログコスト**：ログデータの**CU使用量**に基づいて請求されます。

- **データ転送料**：パプリックインターネット経由のデータ外部転送、同一リージョン内およびクロスリージョンデータ転送に対して請求されます。

詳細については、[監査ログ](./audit-log-cost)および[データ転送](./data-transfer-cost)を参照してください。

## よくある質問\{#faqs}

1. **専用クラスターを一時停止した場合、料金は発生しますか？**

    専用クラスターが一時停止されている場合、ベクターデータベースコストは停止しますが、クラスターを削除するまでストレージ料金は継続されます。

1.  **クラスター作成中または一時停止中に請求はされますか？**

    作成中、一時停止中、再開中、または一時停止状態ではベクターデータベースコストは請求されません。ただし、ストレージコストは引き続き適用されます。
