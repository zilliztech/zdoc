# サーラーレスクラスターコスト

Zilliz Cloudのサーバーレスクラスターは、従量課金モデルを使用しており、主に読み取りおよび書き込み操作によって消費されるリソースに対して請求されます。これにより、事前に固定容量をプロビジョニングする必要なく、処理された実際のワークロードに対してのみ料金を支払うことができます。

サーバーレスクラスターの総コストは、以下の要素の合計です：

- [読み取り](./serverless-cluster-cost#vector-database-costs-read)および[書き込み](./serverless-cluster-cost#vector-database-costs-write)操作のためのベクターデータベースコスト

- [ストレージコスト](./serverless-cluster-cost#storage-cost)

- [その他のコスト](./serverless-cluster-cost#other-costs)（例：データ転送料）

## ベクターデータベースコスト（書き込み）\{#vector-database-costs-write}

書き込みコストは、[挿入、アップサート、および削除操作](./insert-update-delete)によって消費されるコンピューティングリソースを測定します。

インポートおよび一括挿入操作には料金はかかりません。

### 料金計算\{#cost-calculation}

```bash
ベクターデータベースコスト（書き込み） = vCU単価 x 書き込みvCU使用量
```

- **vCU単価：** 100万vCUにつき$4。

- **書き込みvCU使用量：** 書き込み操作に関与するデータサイズに基づいて計算されます。

### 例\{#example}

以下の表は、特定の量のデータをサーバーレスクラスターに書き込む際のvCU使用量とコストのクイックリファレンスチャートを提供します。

     <th><p><strong>書き込みコスト</strong></p></th>
   </tr>
   <tr>
     <td><p>100万個の128次元ベクター</p></td>
     <td><p>0.125</p></td>
     <td><p>$0.5</p></td>
   </tr>
   <tr>
     <td><p>100万個の768次元ベクター</p></td>
     <td><p>0.75</p></td>

   </tr>
   <tr>
     <td><p>100万個の1536次元ベクター</p></td>

   </tr>

**上記表のデータサイズにはスカラーは含まれていません。*

大規模なデータセットでは、vCU使用量とコストを比例的にスケールアップするだけです。たとえば、1000万個の768次元ベクターを書き込むと、約7.5百万vCUが使用され、約$30のコストが発生します。

書き込みvCU使用量とコストの正確な計算については、以下のメトリクスを参照してください：

<table>
   <tr>
     <th><p><strong>操作</strong></p></th>
     <th><p><strong>vCU使用量</strong></p></th>
   </tr>
   <tr>
     <td><p>挿入</p></td>
     <td><p>挿入されたデータ1KB = 0.25 vCU</p></td>
   </tr>
   <tr>
     <td><p>削除</p></td>
     <td><p>削除されたエンティティ1個 = 1 vCU</p><p>存在しないエンティティを削除しても1 vCUが消費されます。</p></td>
   </tr>
   <tr>
     <td><p>アップサート</p></td>
     <td><p>更新されたデータのサイズと削除されたエンティティの数に基づいて計算されます。</p><p>存在しないエンティティを削除しても1 vCUが消費されます。</p></td>
   </tr>
</table>

たとえば、3GB（3,145,728KB）のエンティティをサーバーレスクラスターに挿入し、その後100,000つのエンティティを削除したとします。

- `挿入操作vCU使用量 = 3,145,728 x 0.25 = 78,643 vCU`

- `削除操作vCU使用量 = 100,000 x 1 = 100,000 vCU`

- `総vCU使用量 = 1,000 + 78,643 = 178,643 vCU`

- `合計ベクターデータベースコスト（書き込み） = 0.178643 x 4 = $0.72`

## ベクターデータベースコスト（読み取り）\{#vector-database-costs-read}

このコスト項目は、[検索、ハイブリッド検索、およびクエリ操作](./search-query-get)によって消費されるリソースを測定します。

### 料金計算\{#cost-calculation}

```bash
ベクターデータベースコスト（読み取り） = vCU単価 x 読取りvCU使用量
```

- **vCU単価：** 100万vCUにつき$4

- **読み取りvCU使用量：** 以下の3つの因子に依存します。

    - 検索またはクエリ要求の数：検索またはクエリを実行すればするほど、vCU使用量が高くなります。

    - 各つの検索またはクエリでスキャンされるデータのサイズ：スキャンされるデータが多いほど、vCU使用量が高くなります。

        *ヒント：検索またはクエリの際、Zilliz Cloudはクラスター内のコレクション全体をスキャンします。検索またはクエリの際に[パーティションキー](./use-partition-key)をフィルターとして使用する場合、Zilliz Cloudは指定されたパーティションキーに一致するコレクションの一部のみをスキャンするため、全体の読み取りvCU使用量を下げることができます。*

    - 各つの検索またはクエリで返されるデータのサイズ：返されるデータが多いほど、vCU使用量が高くなります。たとえば、検索でベクターフールを含むすべてのフィールドを返すと、IDフィールドのみを返す検索よりもはるかに多くのvCUが消費されます。

    <Admonition type="info" icon="📘" title="注意">

    <p>各読み取り操作には最小6 vCUの料金がかかります。</p>

    </Admonition>

## ストレージコスト\{#storage-cost}

ストレージコストはベクターデータベースコストとは別に請求され、以下に依存します：

- クラスターリリジョン、クラスタータイプ、およびプロジェクトプラン

- ストレージ使用量

詳細については、[ストレージ](./storage-cost)を参照してください。

## その他のコスト\{#other-costs}

アドオン機能は別途請求されます。アドオンコストの発生源は以下の通りです：

- **データ転送料**：パプリックインターネット経由のデータ外部転送およびクロスリージョンデータ転送に対して請求されます。

詳細については、[データ転送](./data-transfer-cost)を参照してください。
